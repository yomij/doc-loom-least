# Challenge Prompts

Use these only as inspiration. Do not ask them as a fixed checklist.

## Goal And Scope

- What must be true for this to be worth doing?
- Which non-goal would be most tempting to accidentally include?
- What is the smallest version that still proves the point?

## Evidence

- Which claim depends on the weakest evidence?
- Which fact could code, tests, or authority docs answer directly?
- Which historical document might be misleading if treated as current?

## Design

- What boundary will be hardest to change later?
- Which abstraction is doing real work, and which one is ceremony?
- What would break if the project doubled in size?

## Testing

- What regression would the current test strategy fail to catch?
- Is the test proving behavior or proving implementation detail?
- What is the first command that should fail before implementation?

## Documentation Governance

- Is this a long-term fact, a case-local observation, or a derived view?
- What is the source of truth for this claim?
- What needs explicit confirmation before it becomes authority?
