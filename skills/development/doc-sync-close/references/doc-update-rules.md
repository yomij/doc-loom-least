# Document Update Rules

| Layer | Rule |
|---|---|
| L1 Authority | Proposal by default; apply only a confirmed narrow patch. |
| L2 Operational | Update with task evidence. |
| L3 Derived | Mechanical traceable sync only. |
| L4 Historical | Do not change by default. |
| L5 Scratch | Clean/promote only with source handling. |

A narrow authority patch must target an existing boundary, synchronize a small
confirmed fact from plan/execution/tests/diff/user evidence, create no new area,
move/archive/bridge nothing, and involve no conflict, unresolved high-risk
choice, or lifecycle migration. Otherwise propose governance/new work.

Mechanical L3 includes verified links, paths, status rows, commands, parameters,
or short reproducible troubleshooting. Product goals, architecture/workflow
principles, best-practice generalization, and judgment-heavy tutorials remain
proposals.

Authority tables use `Doc | Status | Change | Evidence | Risk | Next`, where
status is proposed/applied and next is none/confirm/governance/new case. Active
lifecycle statuses are only `active`, `draft`, `superseded`, and `archived`;
high-impact docs never become active by agent assertion alone.
